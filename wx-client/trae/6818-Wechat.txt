## 项目简介

本项目是一套移动端 + 云端 + 嵌入式硬件的跨平台远程智能交互控制系统，采用分层架构设计：前端为微信小程序交互界面，后端基于 Python Flask 搭建轻量化 Web 服务，底层硬件搭载粤嵌 6818 嵌入式 Linux 开发板。

项目打通了小程序客户端、服务器、嵌入式开发板三者之间的 Socket 网络双向通信链路，用户仅需通过手机微信小程序，即可随时随地远程下发指令，操控远端 6818 开发板完成多媒体播放、系统壁纸自定义更换、音视频文件上传与自动化压缩处理等一系列硬件交互操作，实现了从移动端可视化操作到嵌入式底层硬件执行的完整技术闭环。

## 核心技术栈

前端交互端：微信小程序原生开发，原生 WXML/WXSS 页面布局，按钮交互、网络请求、文件上传能力

后端服务端：Python Flask 框架，搭建本地 Web 接口服务，接收小程序上传的图片、音频、视频文件，配套图像处理、音频压缩、视频处理脚本，自动完成媒体格式适配与压缩，全局消息缓存，实现指令中转、数据持久化

嵌入式硬件端：粤嵌 6818 开发板（Linux C 开发）C 语言 Socket 网络编程，长轮询拉取服务器控制指令，帧缓冲图形显示驱动开发，音视频播放器调用、系统壁纸设置底层驱动，HTTP 协议数据解析、多任务并发处理

## 项目整体流程与设计

### 前端交互端：微信小程序

微信小程序这里，设计了六个界面（home、open_wallpaper、open_video、open_audio、cmd、message）

功能分别为（主页面、设置壁纸、设置视频、播放音频、命令行、消息列表），每个页面包换四个文件.js、.wxml、.wxss、.json

主页面设计一个轮播图，在打开小程序，监听页面第一次加载时发送http请求，从服务器端获取图片，主页面除了设置壁纸、设置视频、播放音频按钮外，还包含一个语音输入按钮，用户长按唤起录音逻辑，在这里，我一开始想的是调用大模型api来转化我的语音，在搜索资料后，我寻找到了更好的方法：微信内置的同声传译插件，引入该插件后，能够直接在小程序端将语音转为文字，再发给服务器。

在设置壁纸页面，用户可以从手机相册选择图片，点击上传后，小程序会将图片发送到服务器，服务器自动进行格式适配和压缩，然后开发板会拉取并设置为系统壁纸。整个过程操作简单直观，用户只需几步就能完成远程壁纸更换。

设置视频页面则支持用户上传本地视频文件，还可以自定义视频在开发板上的播放时长。上传完成后，视频会被服务器处理并存储，开发板会按照设定的时间自动循环播放，给用户带来更丰富的视觉体验。

播放音频页面不仅支持上传音频文件，还内置了录音功能。用户可以直接在小程序里录制语音，预览确认后再上传到开发板播放，这种设计特别适合临时发送语音消息或指令的场景。

命令行页面是为技术用户准备的高级功能，用户可以直接输入Linux终端命令，发送到开发板执行后，结果会实时回传到小程序界面。这让开发者可以远程调试开发板，无需物理连接就能完成大部分配置工作。

消息列表页面则记录了所有操作历史，包括文件上传、删除、命令执行等，还会显示开发板的报警信息。这样用户可以随时查看系统状态，了解设备的最新动态。

### 后端服务端：Python Flask

后端服务采用轻量级的Flask框架，主要负责处理小程序的文件上传请求，对音视频和图片进行处理，以及管理设备状态和消息传递。服务器会自动对上传的媒体文件进行格式转换和压缩，确保开发板能够流畅播放。同时，服务器还维护着一个消息队列，用于在小程序和开发板之间传递控制指令和状态更新。

### 嵌入式硬件端：粤嵌6818开发板

6818开发板作为系统的核心执行单元，运行着基于Linux的自定义系统。它通过Socket网络与服务器保持长连接，定时轮询获取最新指令。开发板上实现了多种硬件驱动，包括：

- 帧缓冲驱动：用于显示壁纸和视频
- 音频驱动：用于播放音频文件
- 触摸屏驱动：支持用户直接在开发板上进行触摸操作
- 电子画板功能：允许用户在触摸屏上手写绘图

开发板的软件架构采用多线程设计，一个线程负责网络通信，一个线程负责媒体播放，还有一个线程处理用户输入，确保系统运行流畅。

## 技术亮点

1. **跨平台通信**：实现了微信小程序、服务器、嵌入式开发板三者之间的无缝通信，打破了不同设备间的技术壁垒。

2. **实时语音控制**：利用微信同声传译插件，实现了语音指令到文字的实时转换，让用户可以用自然语言控制硬件设备。

3. **媒体处理自动化**：服务器端自动处理上传的音视频和图片，确保格式兼容且大小合适，提升了用户体验。

4. **远程调试能力**：通过命令行界面，用户可以远程执行终端命令，实现对开发板的完全控制和调试。

5. **多线程并发处理**：开发板采用多线程设计，保证了网络通信、媒体播放和用户输入的同时处理，系统响应更迅速。

## 使用方法

1. **准备工作**：确保开发板已连接网络，服务器正常运行。

2. **小程序操作**：
   - 打开微信小程序
   - 在主页面选择需要的功能（设置壁纸、上传视频、录制音频等）
   - 按照界面提示完成操作

3. **查看状态**：在消息列表页面查看操作历史和设备状态。

4. **远程调试**：在命令行页面输入Linux命令，查看执行结果。

## 项目价值

本项目展示了如何构建一套完整的跨平台智能交互控制系统，将移动应用、云计算和嵌入式硬件有机结合。通过微信小程序这一普及的移动平台，降低了用户使用门槛，让普通用户也能轻松控制嵌入式设备。

同时，项目的分层架构设计和模块化实现，为类似系统的开发提供了参考。无论是智能家居控制、远程设备管理还是工业自动化场景，都可以借鉴本项目的技术方案。

未来，我们可以在此基础上扩展更多功能，比如添加更多传感器支持、实现更复杂的自动化场景、接入更多智能设备等，进一步提升系统的实用性和智能化水平。
